# corpus/ — 여기에 옛 글을 넣는다

Phase 0의 토대. **활성도와 분리된 '가치'**로 잊힌 글을 끌어올리려면,
먼저 그 잊힌 글들이 여기 모여 있어야 한다.

> ⚠️ 이 폴더의 **내용물은 git에 커밋되지 않는다** (`.gitignore` 처리).
> SIES는 public 저장소이므로 개인 글은 로컬에만 남는다.
> 폴더 구조(`.gitkeep`)와 이 README만 추적된다.

## 어디에 무엇을 넣나

| 폴더 | 넣을 것 |
|---|---|
| `brunch/` | 브런치 export (글 단위 마크다운/HTML/텍스트) |
| `essays/` | 긁적긁적 에세이, 긴 글 |
| `poems/` | 시, 짧은 운문 |
| `notes/` | 옛 메모, 단상, 메모앱 export |
| `claude-logs/` | Claude/LLM 대화 로그 |
| `misc/` | 위에 안 맞는 나머지 |

## 형식

- **선호:** 마크다운(`.md`) 또는 일반 텍스트(`.txt`).
- HTML/JSON/기타도 일단 넣어도 된다 — 수집·변환 스크립트가 마크다운으로 정규화한다.
- 한 글 = 한 파일이 이상적. 한 파일에 여러 글이 뭉쳐 있으면 나중에 분할 처리한다.
- 파일명/폴더에 날짜가 있으면 *활성도(최근성)* 계산에 쓰이므로 살려두면 좋다
  (예: `2019-03-01-제목.md`). 없으면 파일 mtime으로 대체한다.

## 정직한 점검 (PLAN Phase 0)

> 진짜 '잊힌' 텍스트가 **수백 편 이상**인가?
> 부족하면 이 단계는 *발굴(디지털화·로그 수집)*부터가 된다.

넣은 뒤 글 개수를 세어 이 가설을 점검한다.
