# 망각된-가치 검색기 — 개발 로드맵 (PLAN.md)

> 한 줄 정의: 내 글 코퍼스에서, **활성도와 분리된 '가치'**를 일차 신호로 삼아
> **잊었지만 통찰 있는 과거의 나**를 끌어올리는 개인용 검색 엔진.
> 시장의 '검색/정리'가 아니라 '복구'를 만든다.

---

## 0. 시작 전 못 박을 3가지

**원칙 1 — 80%는 포크, 20%만 직접.**
청킹·임베딩·벡터저장·로컬 UI는 풀린 문제. 직접 만들지 말 것.
직접 짜는 것은 단 둘: **(a) 역전 재순위기, (b) 래칫 루프.**
출발 토대 후보: Reor 포크 / Obsidian + 스크립트 / sqlite-vec·pgvector 직접 조립.

**원칙 2 — AI는 검색 결정에 절대 닿지 않는다.**
- 검색 점수 = 결정론적 산수(벡터 + 가중치). **여기가 제품의 뇌.**
- LLM = 분해·태깅·해설·압축 등 *앞뒤 단계*에만. (이유: LLM은 매끄러운 연결을 고르고, 그게 곧 피하려는 거울이다.)

**원칙 3 — 킬 테스트를 먼저 박는다.**
이 프로젝트 전체가 걸린 단 하나의 가설:
> "내 코퍼스에서 `중간대역 × 저활성` 검색이 그냥 유사도 검색보다
> 잊힌 통찰을 더 잘 끌어올린다."
Phase 1에서 한 달 정직하게 로그를 쌓아 베이스라인을 못 이기면 → **접거나 피벗.**

---

## Phase 0 — 코퍼스 + 한국어 임베딩 검증
**목표:** 토대가 성립하는지 확인 (못 넘으면 위 전부 무의미)

- [ ] 브런치 export, 긁적긁적 에세이, 시, 옛 노트, Claude 대화 로그를 마크다운으로 수집
- [ ] **정직한 점검:** 진짜 '잊힌' 텍스트가 수백 편 이상인가? 부족하면 이 단계가 *발굴(디지털화·로그수집)*부터가 된다
- [ ] 한국어 임베딩 2~3종 벤치 (OpenAI 3-large / Voyage / BGE-m3·KURE 등 한국어 특화)
  - 감각어·함축 많은 네 문장에서 의미검색이 실제로 닿는지 눈으로 확인
- [ ] 저장소 선택: **sqlite-vec**(가벼움) 또는 **pgvector**(SQL 강점)

**산출물:** 코퍼스 폴더 + 임베딩 파이프라인 + 벡터 DB
**완료 기준:** 한국어 의미검색이 "그럴듯하게" 작동함을 확인

---

## Phase 1 — 역전 재순위 ★ 전부를 건 실험
**목표:** 킬 테스트 통과 여부 판정

- [ ] 타임스탬프로 *조잡한 활성도* 계산 (일단 최근성만, 빈도는 나중)
- [ ] 벡터검색 위에 산수만 얹기 — ML 없음:
  ```
  점수 = 관련성 × (1 − 활성도) × 밴드패스마스크
  밴드패스: 유사도 상위 15% 버림(뻔함) + 하위 버림(잡음), 60~85 퍼센타일만 후보
  ```
- [ ] **A/B 하니스:** 베이스라인(순수 유사도) vs 역전. 판정자는 나.
- [ ] 결과 로그: 후보마다 "적중(잊은 통찰이 꽂힘) / 헛것" 표시

**산출물:** 검색 CLI + A/B 로그
**완료 기준 (= 킬 테스트):** 한 달 로그에서 역전이 베이스라인보다 '적중률' 높음
→ 통과하면 진행, 실패하면 중단/피벗

---

## Phase 2 — 일일 루프 + 래칫
**목표:** 매일 작동하는, 그 자체로 평생 쓸 도구

- [ ] **GitHub Actions**가 매일/매주 최근 글(또는 시드)을 받아 잊힌 노트 3~5개를 추출
- [ ] 결과를 이메일 또는 Obsidian 노트로 떨굼
- [ ] 내 반응(보관 / 영감 / 버림)을 기록 → *가치 라벨이 공짜로 누적*
- [ ] 가치 '모델'은 아직 안 만듦. 라벨만 모으고 가중치는 손으로 튜닝

**산출물:** 자동 일일 발굴 파이프라인 + 라벨 로그
**완료 기준:** 손 안 대도 매일 후보가 도착하고, 가끔 나를 놀래킴

---

## Phase 3 — 가치 지표 (싼 것만, 선택)
**목표:** Phase 1이 통했는데 정밀도가 낮을 때만 — 재현율 보완

- [ ] 다리 역할 (betweenness: 서로 다른 군집을 몇 개 잇나)
- [ ] 환원 불가능성 (이웃 노트로 복원 시 오차 클수록 ↑ = 이상치)
- [ ] 쓸 때의 무게 (퇴고·체류·감정강도)
- [ ] 명제 단위 원자화 (LLM이 글을 독립 주장 N개로 분해 → 명제를 임베딩)
- [ ] **장르 인지 밴드패스** (은유적 글은 유사도가 압축 → 밴드를 너그럽게).
      보류된 설계·실데이터 검증: [`docs/genre-aware-bandpass.md`](docs/genre-aware-bandpass.md).
      트리거: 시/소설이 쌓이거나 래칫 라벨 생기면 spread-적응→z-정규화 순.

**완료 기준:** 후보 정밀도가 체감으로 개선

---

## Phase 4 — 그래프층 / 학습된 가치모델 (한참 뒤, 어쩌면 생략)
**목표:** 코퍼스가 충분히 커진 *뒤에야* 본전인 무거운 것들

- [ ] HippoRAG식 PPR + 커뮤니티 검출(Leiden/Louvain) — 규모 커진 후
- [ ] 2단계 라우팅: 군집 분류 → *이웃 군집* 우선 검색 (약한 연결의 금은 옆 동네)
- [ ] 래칫 라벨 수개월치로 "내가 잊었을 때 값지다 여기는 것" 분류기 학습
- [ ] 압축: 중복 노트 N개 → gist 1개 + 포인터 (원본 절대 삭제 금지)

**주의:** **활성도 낮다는 이유로는 절대 삭제 금지.** 삭제는 오직 `가치 낮음 ∧ 중복`.

## 스택 요약

| 역할 | 도구 |
|---|---|
| 오케스트레이션 | Python 스크립트 + cron / GitHub Actions |
| 저장 (네 SQL 강점) | sqlite-vec 또는 pgvector |
| 임베딩 (검색의 의미) | 멀티링구얼/한국어 특화, Phase 0에서 선택 |
| LLM (Claude API) | 분해·태깅·해설·부트스트랩·압축 — **검색 점수엔 절대 안 닿음** |
| 산수 (제품의 뇌) | SQL / numpy — 활성도·밴드패스·랭킹 |
| 프런트 | 안 만듦 → CLI/마크다운 드롭, 필요시 Reor UI 재활용 |

## AI 사용 분업

| 단계 | LLM이 하는 일 | LLM이 안 하는 일 |
|---|---|---|
| 인덱싱 | 명제 원자화, 도메인 태깅, 요지·감정강도 추출 (배치, 1회) | — |
| 검색 | **(없음 — 의도적 공백)** | 후보 선택·순위 결정 |
| 후처리 | 선택된 연결의 *해설*, 비소시에이션 도발 질문 | 후보 거르기·재정렬 |
| 래칫 | 라벨 부족할 때 가치기준 임시 부트스트랩 | 라벨 쌓이면 빠짐 |
| 압축 | gist 요지 추출 | 원본 삭제 |

**비용 규율:** 인덱싱·래칫·압축은 새벽 배치(토큰값 1/n). 해설만 실시간 1회 호출.

## 리스크 / 중단 조건

- **콜드 스타트:** 1일차엔 빈 통. 쐐기는 *임포트*(옛 글 대량 주입)로 day-1 가치를 만든다.
- **킬 테스트 실패:** Phase 1에서 역전이 베이스라인을 못 이기면 중단.
- **한국어 임베딩 한계:** Phase 0에서 의미검색이 안 닿으면 그 위 전부 무의미.
- **스코프 폭주:** 각 Phase는 *그 자체로 내가 쓸 물건*으로 끝나게 잘려 있음. 어디서 멈춰도 남는 게 있다.

---

## 설계 철학 (잊지 말 것)

AI는 잊힌 *재료*를 식탁에 올리고 길을 *언어화*할 뿐, 연결을 대신 *발견*하지 않는다.
통찰의 마지막 도약은 내가 한다. 그 선을 넘으면 이건 또 하나의 똑똑한 거울이 된다.
